[Professor: Pedram Sadeghian | Course: CMSC132]
Course: Despite the name, this is really more 25% Objected-Oriented Programming, 75% Data Structures and Algorithms. So don't go into this course thinking it'll be more of the same stuff as CMSC131. Any of the Java you learn in this course is really taught in order to understand the ideas and implementations of the Data Structures and Algorithms you use throughout the course.
Lectures: I'd say it's very important to attend lecture. Pedram records so there's really no excuse to not go through it, especially since he does a really good job highlighting how the various Data Structures and Algorithms he covers works. He even goes over many of the algorithms that you'll need to implement in the projects step by step, so attending lectures will make the projects much easier.
Discussion/Lab: These were weird because they were more like a second lecture than an actual lab. These are generally miscellaneous Java topics that may or may not appear in a project/test. These are not as worth attending, with the only exception being when the TAs cover sorting algorithms. These are also recorded and you can also watch any TA's recording so if you didn't get a good TA for your section you can just watch the other TA's discussion.
Projects: Most of the projects in this course focus on implementing functionality of data structures/algorithms. Again, attending lecture helps a lot with these projects. Implementing the projects is fairly easy and shouldn't take more than a day or two. The main thing is to make sure you know how to debug your code and write student tests as secret tests are usually weighted around 10-20% of the grade. Main thing I'd recommend is writing student tests as you work on the project, I felt that helped me a lot with making sure my projects were good. Also try to maintain good style and write comments as you go so you don't have to deal with doing it all at the end.
Quizzes: There are 5 quizzes of varying formats. 2 of them were "write code" quizzes. When Pedram asks you to write code he expects you to know how to write an algorithm that solves a problem from scratch. The main way you can practice this is through the practice tests and projects. He recommends you write the algorithm out on paper and then code it up on Eclipse. I'd recommend knowing how to trace out algorithms, pay attention to the specific details of the question, as well as practicing material that relates to what Pedram says the quiz will focus on. Another one was a "debugging" quiz where you sat with a TA and showed them that you can use the debugger. Just know how to use the debugger and it's a free 30 points. Last 2 quizzes are take-home "project" quizzes which simply involved you having to write a code piece on Eclipse, which are also free 30 points each since there are no secret tests with these quizzes and lecture + lab gets canceled that day so you can work on those quizzes.
Exams: Exams generally consist of 3 parts, Fill-In-The-Blank, Short-Answer, and Code Writing. For code writing my advice is the same as the code writing involved in the quizzes. For short-answer questions, I'd recommend knowing the slide material and knowing how to trace out any of the algorithms that you learn in class/implement in projects. Understand how the Java compiler works as that'll be important for understanding some of the concepts you need to know for these exams. For fill-in-the-blanks, again just review the slides thoroughly and use a tool like Quizlet to memorize key terms.
Other Random Advice
- I didn't go to office hours (didn't really need to) but if you are struggling I'd recommend it. The TA's this semester were very good and helpful. If you have small questions, use Piazza, the TA's (and even Pedram at times) will respond quite quickly.
- Pay attention to Pedram's posts on Piazza, especially when he releases details about the Quizzes/Exams. He gives away a lot about what the test will focus on which will help you guide your studying.
- Don't rely on the practice exams for your studying. While they offer some good material for practice they rarely if ever cover everything you'll need to go over for the test. Instead, try to create your own practice questions based on the slide material as well as study how you implemented the projects. If Pedram recommends you do a project before a test then make sure to do that project for practice. I'd also recommend practicing questions on the site Leetcode. Some of the questions are a bit advanced but the site offers good practice for questions related to certain data structures (especially LinkedList and Trees).
Other Information

Exams are on the tougher side but are 100% doable if you pay attention to what Pedram posts about them and make sure to practice all the material instead of just relying on practice exams.
